# 活动专场车源 · 需求评审讲解稿

## 一、背景与目标

本次需求的核心目标是：**在现有专场车源体系上，新增"活动专场"能力**，支持运营人员自由配置专题活动，将特定条件的车源以活动专页形式对外推广，同时在移动端首页、专场车源列表、PC端官网等多个入口进行曝光引流。

整体分为两大部分：
- **后台**：活动配置管理（创建、编辑、启用、停用、统计）
- **前台**：用户侧各端（移动端首页、移动端专场入口、移动端活动专场页、PC端官网）的展示与交互

---

## 二、后台模块

### 2.1 权限范围

活动专场后台菜单，仅开放给**运营**与**管理者**角色。

---

### 2.2 活动列表页

活动列表是后台运营管理活动的核心入口，支持以下核心能力：

**查询与展示**
- 支持按**活动名称**、**启用状态**组合筛选
- 列表字段：活动编号、活动主题、运营类型、起止时间、排序号、创建人、最后操作时间、状态、操作

**活动状态与可执行操作**

| 状态 | 描述 | 可执行操作 |
|---|---|---|
| **草稿** | 新建保存后，未生效 | 编辑、启用、复制、PC端预览、移动端预览 |
| **启用** | 生效中，对外展示 | 编辑、停用、访问、复制链接、复制、查看专场 |
| **停用** | 人工停用或到期自动停用 | 查看、复制 |

**核心操作说明**

- **启用**：点击后弹窗确认，校验是否已配置专场及车源；校验通过后正式发布，系统动态生成对外链接。
- **停用**：弹窗风险告知，确认后停止对外展示，访问链接跳转"活动已结束"异常页。
- **查看专场**（启用态）：按专场Tab展示今日对外车源，支持数据统计。
- **端预览**（草稿态）：支持PC端及移动端静态样式预览。

---

### 2.3 新建 / 编辑活动

新建与编辑分两步：**第一步设置活动信息，第二步设置专场及车源**。

#### 第一步：活动基础信息

| 字段 | 说明 |
|---|---|
| **活动名称** | 必填，限18个中文字符 |
| **车源加入方式** | 固定"系统自动筛选" |
| **活动时间** | 可选"指定时间"（开始/结束时间均必填，结束时间必须大于当前时间）；默认勾选"不对外展示" |
| **展示排序** | 多活动并行时，控制活动间展示顺序 |
| **活动Banner** | 必填，包含移动端Banner、移动端列表图/分享图、PC端Banner图 |
| **活动说明** | 非必填，分PC端与移动端两种格式编辑，配置后可在前台展示 |

**保存规则**
- "**保存**"：保存为草稿，不发布
- "**保存并启用**"：满足必填项条件后，直接发布为"启用"状态

#### 第二步：专场配置与车源筛选

**专场配置**
- 一个活动支持多个专场，每个专场独立配置名称、车源类型（竞价大厅+有保留拍，可多选）、展示排序
- 专场名称**必填**；排序号相同时，保存校验提示冲突
- 支持**增加/删除专场**（第一个专场不提供删除）

**车源筛选条件**（多选值为"或"关系，不同条件之间为"与"关系）

| 条件 | 说明 |
|---|---|
| 品牌车型 | 关键字搜索，多选 |
| 车龄 | 1年内、1-3年、3-5年、5年以上，可多选 |
| 里程 | 1万内、1-3万、3-5万、5-10万、10万以上，可多选 |
| 价格 | 3万内、3-5万、5-8万、8-15万、15-20万、20万以上，可多选 |
| 能源类型 | 汽油、柴油、电力、油电、其它 |
| 排放标准 | 国I～国VI，可多选 |
| 是否新能源 | 按上拍时填写字段筛选 |
| 车况评分/事故等级 | 支持多选 |
| 国别 | 国产、德系、日系等，可多选 |
| 集团 | 按关键字搜索，多选 |
| 车牌所在地 / 所在区域 | 支持省/市多选 |

- "**车源试算**"：弹窗展示前三天该筛选条件下在拍车辆台数，用于数据量评估。
- "**重置条件**"：一键清空所有筛选条件。

**编辑限制（启用状态）**
- 活动定义信息（名称、Banner、专场说明等）**不可修改**
- 仅"**展示排序**"和"**转发文案**"可修改
- 车源筛选条件可调整，保存后**即时生效**到线上
- **不支持**在已启用状态下新增专场

---

### 2.4 活动统计

为运营提供活动效果数据追踪，权限单独配置。

**筛选条件**
- **日期**（必填，默认前7天）
- **选择专场**（动态加载最近启用过的活动及子专场，联动选择）

**统计指标（按日分行展示）**

| 指标 | 定义 |
|---|---|
| 活动列表页UV | 各端每日登录客户访问活动列表的去重数 |
| 点击车源UV | 各端每日从列表点击车源进详情的去重数 |
| 车辆详情页UV | 各端每日访问车辆详情页的去重数 |
| 成交客户数 | 当天通过本专场成交的去重客户数 |
| 结束上拍台数 | 当天在本专场结束上拍的车源台数 |
| 成交台数 | 当天结束上拍车源中成交的台数 |

- **分端统计**：PC、APP、小程序分列展示
- **排序规则**：按日期**正序**
- **导出功能**：按当前查询结果导出Excel，导出权限独立配置

## 三、前台模块

### 3.1 移动端首页改版

首页新增**动态自定义专区**模块，整体分区如下：工具栏 → Banner区 → 固定专场区 → **动态自定义专区（新增）** → 推荐/订阅区。

**新增 —— 我的订阅入口**
- 工具栏新增"**我的订阅**"链接，跳转至"关注订阅"的"我的订阅"Tab页。

**固定专场区调整**
- 固定显示**比亚迪置换车专场**、**报废车专场**、**竞价大厅**入口
- 优先展示比亚迪和报废车专场，若对应专场无车，由竞价大厅占位
- "更多专场拍卖"链接跳转**专场车源首页**

**动态自定义专区（核心新增）**
- 动态展示**已启用**且**时间仍在生效**的自定义活动专场，按排序号升序展示
- **无车源时**：该专场不对外展示；所有专场均无车时，整个专区不展示

**有车源时，根据有车场次数量动态调整版面**：

| 有车专场数量 | 展示规则 |
|---|---|
| **3个及以上** | 每专场展示**快结束的2台车**，支持左右滑动 |
| **仅2个** | 每专场展示**快结束的3台车** |
| **仅1个** | 展示**快结束的5台车** |

- 车辆标注：品牌车系（超5个中文字符加`...`省略）+ 起拍价
- 点击专场区域，跳转对应**活动专场列表页**

**推荐/订阅区优化**
- "为您推荐"与"我的订阅"默认展示台数由8台改为**30台**
- 上滑到底时瀑布流加载，加载完成提示"**已到最后**"

---

### 3.2 移动端专场入口

**首页点击入口**
- 点击活动专场专区入口，进入**活动列表页**

**链接专场停用时的降级处理**
- 若专场已停用/结束或当日无有效上拍车源，展示提示页：
  - 点击"**其它专场车源**"跳至专场车源列表
  - 左上角返回至活动列表
  - 底部提供"**为您推荐**"6辆车（取首页推荐前6辆，按首页顺序展示）

**专场车源列表中的活动专场展示规则**
- **展示条件**：启用中且当日有上拍车源的专场才展示
- **展示位置**：在政企专场之后轮次插入
  - 多个活动同时进行时，仅第1个活动专场插在第1个政企专场后，其余排在政企专场最后
- **时间展示**：有指定开始时间则显示，未指定则不显示
- **点击交互**：跳转至**活动专场列表页**

---

### 3.3 移动端活动专场页

#### H5活动页

**页面展示内容**
- 动态读取活动开始/结束时间展示（无配置则不展示）
- 有配置活动说明时，显示说明入口；否则不展示
- 专场数量=1时，不显示专场Tab及专场名称，以活动名称整体展示
- 多专场时，Tab过多支持**左右滑动**
- 车辆排序：按指定次序正序；次序相同时按结束时间正序；已流拍/成交车辆**不展示**
- 页面失效处理：若无在拍车源或活动已失效，跳转至"活动结束"展示页；若某专场下车源全部下架，刷新后该专场Tab不再显示

**分享功能**

| 端 | 支持分享方式 |
|---|---|
| 唯车拍APP | 微信好友、朋友圈、复制链接 |
| 唯车拍小程序 | 仅支持好友分享 |

- 外部微信用户通过分享链接可直接访问H5活动页
  - APP分享链接：引导打开/安装唯车拍APP
  - 小程序分享链接：引导使用唯车拍小程序访问

**访问统计**：访问活动专场时，系统记录访问日志用于统计。

#### APP活动专页（H5嵌套引导层）

- 顶部固定APP导航层，提供"**打开APP**"按钮
- 每个专场最高加载前**6辆车**；展示完6辆后，底部提供打开APP按钮引导
- 触发"打开APP事件"的场景：点击打开APP按钮、查看更多车源、关注、查看车辆详情等
  - 已安装：引导打开并访问指定活动页
  - 未安装：引导至下载页（安卓→腾讯应用宝或第三方浏览器；iOS→App Store）

---

### 3.4 PC端活动专场

**官网入口展示规则**（同移动端一致）
- 启用中且有上拍车源的专场才展示
- 多活动时，第1个插在第1个政企专场后，其余排在最后
- 有开始时间则展示，否则不展示
- 点击跳转**活动专场列表页**

**专场列表页**

竞价大厅新增"**专场活动**"板块，支持：
- 顶部路径显示当前活动名称
- 动态展示：活动时间（有配置则显示）、Banner、活动说明、分享按钮
- 单专场时不显示场次Tab切换；多专场时按配置顺序切换Tab
- **活动说明**：有配置时显示入口，点击弹窗展示，支持点击关闭或点击弹窗外部区域关闭
- **分享活动**：每个活动右侧提供分享按钮，鼠标悬停显示二维码，手机微信扫码访问APP活动页，支持进一步分享
- **车源展示**：同竞价大厅既有样式，字段包含查验标、分值、城市、车型、注册日期、里程、排放标准、车牌归属、过户次数、起拍价、结束时间等

**专场结束/停用状态**

若专场停用、时间结束或无有效在拍车源时：
- 顶部动态显示活动路径名称、Banner、活动时间
- 展示"活动已结束"提示，**不提供分享功能**
- 点击"**其它专场车源**"跳至专场车源列表
- 底部"为您推荐"提供**8辆车**（取数逻辑同捡漏专场）

## 四、关键设计决策说明

| 决策点 | 设计选择 | 原因 |
|---|---|---|
| 车源加入方式 | 固定"系统自动筛选" | 降低运营手工维护成本，车源实时动态更新 |
| 活动时间默认"不对外展示" | 适配长期专场（如新能源专区）无时间限制的场景 | 兼顾短期活动与长期专场两种诉求 |
| 首页动态专区版面按有车场次数动态变化 | 保证展示内容充实，避免版面稀疏 | 提升用户浏览体验 |
| 启用状态下活动定义信息不可修改 | 防止已曝光的活动信息被随意篡改 | 保障对外展示一致性 |
| PC端分享生成二维码引导APP访问 | 利用PC入口导流至APP端 | 提升APP活跃度 |

## 五、待确认事项

> 以下为评审时建议同步确认的开放问题：

1. **专场数量上限**：新建活动时"一个活动支持最多 X 个专场"，当前X尚未填写，需明确值。
2. **"XX拍"名称**：PRD多处出现"XX拍"，评审时需确认正式名称。
3. **首页动态专区起拍价字段**：建议确认起拍价展示格式（是否带"万"单位、是否需处理超长数字）。
4. **活动统计访问UV统计口径**：未登录用户是否也统计在内，PRD目前定义为"登录客户数量"，需与数据团队对齐。
